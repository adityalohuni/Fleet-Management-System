# Documentation

This folder contains reference documentation for the Fleet Management System. It is intended to be a source of truth for AI assistants and developers working on the project.

## Contents

- [Architecture & Requirements](./ARCHITECTURE.md): Detailed overview of the system architecture, backend requirements, data models, and deployment strategies.
- [Backend Architecture](./BACKEND_ARCHITECTURE.md): Specifics of the backend layered architecture.
- [Backend Plan](./BACKEND_PLAN.md): Implementation plan and progress.
- [Frontend Integration Plan](./FRONTEND_INTEGRATION_PLAN.md): Strategy for connecting the React frontend to the Rust backend.
- [API Layer](./API_LAYER.md): Documentation for the API layer, including how to access the Swagger UI.
- [Service Layer](./SERVICE_LAYER.md): Documentation for the business logic layer.
- [Repository Layer](./REPOSITORY_LAYER.md): Documentation for the data access layer.
- [Database Schema](./DATABASE_SCHEMA.md): SQL schema definitions.
- [Testing Strategy](./TESTING_STRATEGY.md): Approach to testing the application.
- [MVP & Extensions](./MVP_AND_EXTENSIONS.md): Converted documentation from PDF detailing the MVP scope, architecture, and future extension roadmap.
